# Claude 명령어: Deploy Check

배포 전 필수 점검 항목을 순서대로 실행합니다.

## 사용법

```
/deploy-check
```

## 프로세스

1. **린트 검사** — `npm run lint` 실행
   - 코드 스타일 및 잠재적 오류 확인
2. **프로덕션 빌드** — `npm run build` 실행
   - TypeScript 타입 에러 포함 전체 빌드 검사
3. 결과에 따라 안내:
   - ✅ 이상 없음 → 배포 준비 완료 메시지
   - ❌ 에러 있음 → 에러 원인과 해결 방법 안내

## 점검 항목

| 단계 | 명령어 | 확인 내용 |
|------|--------|---------|
| 1단계 | `npm run lint` | ESLint 규칙 위반, 코드 품질 |
| 2단계 | `npm run build` | TypeScript 타입 에러, 빌드 오류 |

## 자주 나오는 에러 유형

- **TypeScript 에러**: 타입이 맞지 않는 경우 → 에러 위치와 수정 방법 안내
- **ESLint 경고**: 사용하지 않는 변수, import 등 → 해당 줄 수정 안내
- **빌드 실패**: 누락된 파일, 잘못된 경로 등 → 원인 분석 후 해결책 제시

## 참고사항

- 빌드 성공이 곧 배포 가능 상태를 의미하지 않습니다
- 실제 배포는 Vercel 대시보드 또는 `vercel` CLI를 사용하세요
- 빌드 결과물은 `.next/` 폴더에 생성됩니다 (git에 포함하지 않음)
